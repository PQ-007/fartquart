## Директорийн бүтэц

```
content/
  blog/          → /blog дээр гарна
  notes/         → /notes дээр гарна
    [номын нэр]/
      index.md   → номын үндсэн хуудас
      1章 ....md  → бүлэг тус бүр
  creations/     → /creations дээр гарна
```

---

## Label — агуулгын төрөл

`label` талбар нь агуулгыг хаана харуулахыг тодорхойлно.

| Label | Хаана гарна | Файл байршил |
|---|---|---|
| `essay` | /blog | content/blog/ |
| `note` | /blog | content/blog/ |
| `lesson-note` | /blog | content/blog/ |
| `internship` | /blog | content/blog/ |
| `project-log` | /blog | content/blog/ |
| `contest` | /blog | content/blog/ |
| `book-review` | /blog | content/blog/ |
| `book-note` | /notes | content/notes/[нэр]/index.md |

---

## Blog пост — frontmatter

```md
---
title: Гарчиг
date: 2026-01-15
label: essay
description: Товч тайлбар (карт дээр харагдана)
tags:
  - TagNэр
draft: false
---
```

`draft: true` тавьвал сайтад харагдахгүй, хөгжүүлэхэд тохиромжтой.

## Book review — номын шүүмж

`/blog` дээр гарна. Нүүр зураг, үнэлгээ, хуудасны тоо автоматаар харагдана.

```md
---
title: Clean Code
date: 2026-01-15
label: book-review
author: Robert C. Martin
rating: 4
genre: Software Engineering
pages: 431
description: Товч тайлбар
tags:
  - Engineering
---
```

`rating` нь 1–5 хооронд байна. Нүүрлэл хуудсанд одны дүрслэл болж харагдана.

---

## Book note — номын тэмдэглэл (бүлэгтэй)

`/notes` дээр гарна. `content/notes/[номын нэр]/` хавтас дотор байрлана.

**index.md** — номын ерөнхий хуудас:

```md
---
title: ゼロから作る Deep Learning №1
date: 2025-09-13
label: book-note
author: 斎藤 康毅
rating: 5
genre: Deep Learning
pages: 318
tags:
  - Python
  - JP
---
Номын тухай товч танилцуулга...
```

**Бүлгийн файл** — `1章 нэр.md` гэх мэт:

```md
---
title: 1章 — Гарчиг
date: 2025-10-01
tags:
  - Book/DeepLearning1
new-word:
  - word: パーセプトロン
    definition: Хиймэл нейроны анхдагч алгоритм.
  - word: 閾値 — Threshold
    definition: Нейрон галлах босго утга.
draft: false
---
```

`new-word` талбар нь баруун талын sidebar-т "New Words" хэсэгт харагдана. Формат хоёр янз байж болно:

```yaml
new-word:
  - word: Үг
    definition: Тодорхойлолт.
  - "Үг: Тодорхойлолт"
```

Бүлгийн хооронд автоматаар **← Өмнөх / Дараах →** навигаци үүснэ. Sidebar дотор дэлгэрэнгүй гарчгийн жагсаалт, контент дахь гадаад линкүүд **Sources** болж харагдана.

## Creation — бүтээл

`/creations` дээр гарна. `.md` эсвэл `.mdx` хоёулаа болно.

```md
---
title: Ivo
description: Японы толь бичиг апп
tags: [mobile, swift, ios]
date: 2026-06-01
cover: creations/ivo/cover
demo: https://example.com
repo: https://github.com/user/ivo
youtube: dQw4w9WgXcQ
draft: false
---
```

## Код блок

Стандарт markdown fence ашиглана — `CodeSnippet` компонент цаашид хэрэглэхгүй.

**Зөвхөн хэл зааж өгөх:**

````md
```python
print("Hello")
```
````

**Файлын нэрээр label харуулах:**

````md
```Character.py
class Character:
    def __init__(self):
        self.health = 100
```
````

Өргөтгөлөөс хэлийг автоматаар тодорхойлно: `.py` → Python, `.ts` → TypeScript, `.rs` → Rust гэх мэт.

Дэмжигдсэн хэлүүд: `tsx`, `ts`, `jsx`, `js`, `css`, `html`, `json`, `bash`, `python`, `rust`, `go`, `java`, `c`, `cpp`

## MDX компонентууд

`.mdx` өргөтгөлтэй файлд ашиглана.

**YouTube:**
```mdx
<YouTube id="dQw4w9WgXcQ" />
```

**Видео файл:**
```mdx
<Video src="/videos/demo.mp4" />
```

**Математик томьёо** (LaTeX):
```md
$$
y = \begin{cases} 0 & (x \leq \theta) \\ 1 & (x > \theta) \end{cases}
$$
```

Inline: `$x_1 + x_2$`

## Зураг

Markdown стандарт синтакс:

```md
![Тайлбар](/images/example.png)
```

Зургийг дарвал lightbox-д томоор харагдана.

## Tags

`tags` массив дахь утгуудаар `/tags` хуудас дээр графт харагдана. Label-ийн нэр автоматаар tag болон тооцогдоно. Том жижиг үсэг ялгана — `Engineering` ба `engineering` өөр tag.

---

## Контент нэмэх товч дараалал

1. Файл үүсгэх (байршлыг дээрх хүснэгтээс харна)
2. Frontmatter бичих — `title`, `date`, `label`, `description` заавал
3. `draft: true` тавьж эхлэх
4. Бичиж дуусаад `draft: false` болгох
